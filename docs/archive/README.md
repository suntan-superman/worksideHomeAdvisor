# Archive

This folder preserves documents that still have historical value but are no longer the implementation source of truth.

## Archive Rules

- keep documents here for reference, not for default planning
- prefer `docs/LIVE_SYSTEM.md` for current product direction
- prefer `docs/VISION_TIERS.md` for current vision scope
- prefer phase-specific execution docs only when they are newer and clearly aligned with the live codebase

## Current Archive Groups

### `legacy-foundation/`

Early scaffold and schema documents from the original Prisma/Postgres-oriented planning phase.
These are intentionally retained because they explain the product's earlier shape, but they should not drive new implementation decisions.
