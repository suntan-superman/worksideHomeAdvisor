# Consequence Framing Examples

Consequence framing is generated in:

- `report.payload.consequenceFraming.summary`
- `report.payload.consequenceFraming.lines[]`

Pattern used:

- Professional and credible language
- Non-alarmist wording
- Cautious risk framing ("may", "can", "could")

Examples now used:

- "3 photo retake recommendations remain, which may weaken first impressions and reduce early buyer appeal."
- "5 checklist items remain open and may delay launch momentum if unresolved."
- "Pricing still needs confirmation, which may limit value perception or create negotiation friction."
- "Current readiness is 58/100, which may leave presentation value on the table until top prep items are completed."
- "Execution support is still limited, which may slow prep completion and showing readiness."
