# Action Plan Scaling

## Depth by seller-report class
- `prep_report`: 4 top actions, 3 action cards, support pages enabled
- `balanced_report`: 3 top actions, 2 action cards, support pages enabled
- `launch_report`: 2 top actions, 1 action card, support pages disabled unless blockers remain

## Goal
Action depth matches readiness state so near-launch properties stay concise while prep-heavy properties keep guidance depth.
