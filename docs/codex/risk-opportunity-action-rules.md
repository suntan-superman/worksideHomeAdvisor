# Risk, Opportunity, Action Rules

## Editorial separation
- Risk: current friction suppressing response now
- Opportunity: highest-value upside unlocked by targeted fixes
- Action: next concrete execution step

## Implementation notes
- `deriveReadinessRiskOpportunity(...)` produces distinct risk/opportunity framing.
- Action layer is driven by top prioritized checklist/action cards.
- Readiness economics now includes explicit action and risk reminder lines to avoid duplicate phrasing.

## Intended behavior
Risk and opportunity should be related, but not rewritten copies of the same sentence.
